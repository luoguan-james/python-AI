# 技术架构设计文档

## 1. 项目概述

### 1.1 项目简介
本项目是一个基于 FastAPI 和 Scrapy 的爬虫系统，提供 RESTful API 接口进行数据管理和消息队列操作。系统采用分层架构设计，包含 API 层、服务层和数据层。

### 1.2 技术栈
| 组件 | 技术选型 | 版本 |
|------|----------|------|
| 运行时 | Python | 3.9+ |
| Web 框架 | FastAPI | 0.100+ |
| 爬虫框架 | Scrapy | 2.9+ |
| ORM | SQLAlchemy | 1.4+ |
| 消息队列 | 内存/文件/Redis | - |
| 认证 | JWT (Bearer Token) | - |

## 2. 系统架构

### 2.1 分层架构
```
┌─────────────────────────────────────────┐
│              API 层 (src/api/)           │
│  ┌─────────────┐  ┌──────────────────┐  │
│  │ Controllers  │  │   Middleware     │  │
│  │ (auth/user)  │  │ (auth/error)     │  │
│  └──────┬──────┘  └──────────────────┘  │
├─────────┼───────────────────────────────┤
│         ▼                                │
│  ┌──────────────────────────────────┐    │
│  │        服务层 (src/services/)     │    │
│  │  ┌──────────┐  ┌──────────────┐  │    │
│  │  │AuthService│  │ UserService  │  │    │
│  │  └─────┬────┘  └──────┬───────┘  │    │
│  │        │              │          │    │
│  │  ┌─────┴──────────────┴───────┐  │    │
│  │  │    Exceptions (领域异常)    │  │    │
│  │  └────────────────────────────┘  │    │
│  └──────────────────────────────────┘    │
├──────────────────────────────────────────┤
│         ▼                                │
│  ┌──────────────────────────────────┐    │
│  │        数据层 (src/repositories/) │    │
│  │  ┌────────────────────────────┐  │    │
│  │  │    UserRepository          │  │    │
│  │  └────────────────────────────┘  │    │
│  └──────────────────────────────────┘    │
├──────────────────────────────────────────┤
│         ▼                                │
│  ┌──────────────────────────────────┐    │
│  │    数据模型 (src/models/)        │    │
│  │  ┌────────────────────────────┐  │    │
│  │  │    User (dataclass)        │  │    │
│  │  └────────────────────────────┘  │    │
│  └──────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

### 2.2 数据流
```
客户端 → API 控制器 → 服务层 → 仓储层 → 内存存储
         ← JSON 响应 ← DTO/Model ←
```

## 3. 模块详细设计

### 3.1 API 层 (`src/api/`)

#### 3.1.1 控制器
| 控制器 | 路由前缀 | 功能 |
|--------|----------|------|
| `auth_controller.py` | `/auth` | 用户认证（登录） |
| `user_controller.py` | `/users` | 用户管理（查询、创建） |

#### 3.1.2 中间件
| 中间件 | 功能 |
|--------|------|
| `auth_middleware.py` | JWT Token 验证，提取当前用户 |
| `error_handler.py` | 统一异常处理，将领域异常映射为 HTTP 状态码 |

### 3.2 服务层 (`src/services/`)

#### 3.2.1 服务组件
| 服务类 | 职责 |
|--------|------|
| `AuthService` | 用户认证逻辑，密码验证，Token 生成 |
| `UserService` | 用户 CRUD 操作，用户名唯一性校验 |

#### 3.2.2 异常体系
| 异常类 | HTTP 状态码 | 说明 |
|--------|-------------|------|
| `DomainException` | 400 | 通用领域异常 |
| `AuthenticationError` | 401 | 认证失败异常 |

### 3.3 数据层 (`src/repositories/`)

#### 3.3.1 仓储组件
| 仓储类 | 操作方法 |
|--------|----------|
| `UserRepository` | `find_by_id()`, `find_by_username()`, `save()`, `delete()` |

### 3.4 数据模型 (`src/models/`)

#### 3.4.1 领域模型
| 模型 | 类型 | 字段 |
|------|------|------|
| `User` | dataclass | `id`, `username`, `email`, `password_hash`, `created_at` |

## 4. 消息队列系统

### 4.1 架构
```
生产者 (Producer) → 队列 (Queue) → 消费者 (Consumer)
```

### 4.2 队列实现
| 实现类 | 后端 | 特性 |
|--------|------|------|
| `InMemoryQueue` | 内存 | 支持优先级、Topic 过滤、批量操作 |
| `FileQueue` | 文件系统 | 持久化到磁盘，支持崩溃恢复 |
| `RedisQueue` | Redis | 分布式队列，支持多进程消费 |

### 4.3 消息生命周期
```
创建 (NEW) → 发送 (PENDING) → 接收 (PROCESSING) → 确认 (DONE)
                                                      → 失败 (FAILED) → 重试
```

## 5. 爬虫数据模型

### 5.1 数据项
| 数据项 | 说明 | 关键字段 |
|--------|------|----------|
| `TsItem` | 教师/学生信息 | `title`, `name`, `email`, `source_url` |
| `CourseItem` | 课程信息 | `course_id`, `course_name`, `capacity`, `enrolled` |
| `ArticleItem` | 文章信息 | `article_id`, `title`, `content`, `tags` |

### 5.2 数据库模型
| 模型 | 对应表 | 说明 |
|------|--------|------|
| `TsModel` | `ts_items` | 教师/学生数据持久化 |
| `CourseModel` | `course_items` | 课程数据持久化 |
| `ArticleModel` | `article_items` | 文章数据持久化 |

## 6. 配置管理

### 6.1 配置项
| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `LOG_LEVEL` | 日志级别 | `INFO` |
| `DATABASE_URL` | 数据库连接字符串 | - |
| `REDIS_URL` | Redis 连接字符串 | - |
| `JWT_SECRET` | JWT 签名密钥 | - |

## 7. 安全设计

### 7.1 认证流程
```
1. 客户端 POST /auth/login 提交用户名密码
2. 服务端验证凭据，返回 JWT Token
3. 客户端在后续请求的 Authorization 头携带 Token
4. 中间件验证 Token 有效性，提取用户信息
```

### 7.2 错误处理
- 所有异常通过 `error_handler.py` 统一处理
- 返回标准错误格式：`{ "error": "...", "message": "...", "status_code": ... }`

## 8. 部署架构

### 8.1 组件部署
```
负载均衡器 → FastAPI 应用实例 (多副本)
                ├── 内存队列 (单机)
                ├── 文件队列 (单机)
                └── Redis 队列 (分布式)
```

### 8.2 环境要求
- Python 3.9+
- 可选：Redis 服务（使用 RedisQueue 时）
- 可选：PostgreSQL 数据库（生产环境）

# 技术架构设计文档

## 4. 消息队列系统

### 4.1 架构
```
生产者 (Producer) → 队列 (Queue) → 消费者 (Consumer)
```

### 4.2 队列实现
| 实现类 | 后端 | 特性 |
|--------|------|------|
| `InMemoryQueue` | 内存 | 支持优先级、Topic 过滤、批量操作 |
| `FileQueue` | 文件系统 | 持久化到磁盘，支持崩溃恢复 |
| `RedisQueue` | Redis | 分布式队列，支持多进程消费 |

### 4.3 消息生命周期
```
创建 (NEW) → 发送 (PENDING) → 接收 (PROCESSING) → 确认 (DONE)
                                                      → 失败 (FAILED) → 重试
```

## 7. 安全设计

### 7.1 认证流程
```
1. 客户端 POST /auth/login 提交用户名密码
2. 服务端验证凭据，返回 JWT Token
3. 客户端在后续请求的 Authorization 头携带 Token
4. 中间件验证 Token 有效性，提取用户信息
```

### 7.2 错误处理
- 所有异常通过 `error_handler.py` 统一处理
- 返回标准错误格式：`{ "error": "...", "message": "...", "status_code": ... }`
